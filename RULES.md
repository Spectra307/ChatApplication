# ChatApplication - Project Rules

## 1. Modification Policy
- **Never modify anything unnecessary without permission**
- All changes must be authorized before implementation
- Request explicit approval before making any code modifications

## 2. Tech Stack

### Frontend
- CSS
- HTML
- JavaScript

### Backend / Core Logic
- Java

### Database
- SQL
- Supabase
- PostgreSQL

## 3. Code Documentation
- Code documentation is **important and mandatory**
- All functions, classes, and complex logic must be properly documented
- Follow consistent documentation standards across the project

## 4. Architecture & Implementation Guidelines
- High-level modifications or implementation of new ideas will be provided through **StarUML scripts**
- Design decisions and architectural changes follow the StarUML diagrams provided
- Reference the UML diagrams before implementing significant features
